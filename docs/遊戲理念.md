# 拉麵怪奇物語 — 遊戲設計理念

> 一款純手作視角、一次只做一碗的直式手機拉麵小品。
> Made with Godot 4.3 ・ 線上試玩：<https://meloalright.github.io/ramen-ya/>

---

## 一、一句話概念

**「親手做一碗拉麵」**——不經營、不排隊、不數錢，只專注於把眼前這一碗麵做好。
玩家站在拉麵店的料理台後，從滾燙的湯鍋舀湯、從麵鍋撈麵、灑上客人要的配料，做對了就端上桌。

整個體驗追求**輕、爽、手感**：拿起、放下、灑料、上菜，每一下點擊都立刻有回饋，沒有多餘的選單與數值。

---

## 二、核心理念

1. **手作感優先**
   每個動作都對應一個真實的料理工具：舀湯是「大湯勺」、撈麵是「大筷子夾著麵條」、灑料是指尖捏一撮。游標拿著什麼、就看得到什麼。

2. **一次一單，做完即結算**
   不做排隊經營。畫面上永遠只有一張訂單，做對就上菜、彈出勝利結算，乾淨俐落。

3. **介面只在需要時出現**
   預設畫面上沒有任何按鈕。
   - 當碗符合訂單 → 才浮出「上菜」。
   - 當灑錯了無法挽回的配料 → 才浮出「倒掉」。
   讓玩家專注在「做」，而不是「找按鈕」。

4. **場景即介面，桌子無限延展**
   料理台的木桌、菜單頁的櫃台，都是程式化繪製、隨螢幕無限鋪滿，任何比例的手機 / 桌面都不留黑邊。

---

## 三、玩法循環

```
菜單（櫃台）→ 點畫面開始 → 做麵 → 上菜成功 → 勝利結算 → 再來一單
```

### 做一碗麵的步驟

| 步驟 | 操作 | 視覺回饋 |
|---|---|---|
| 下湯 | 點下方**湯鍋的蒸汽** | 游標提起一把盛著**黃油湯**的大湯勺；點碗倒入 |
| 煮麵 | 點下方**麵鍋的蒸汽**（第一次） | 麵下鍋開始煮，旁邊出現**火候條** |
| 撈麵 | 再點一次麵鍋 | 游標提起一雙夾著**長白麵條**的大筷子；點碗放入 |
| 加牛肉 | 點**牛肉**小料碗 | 游標拿著牛肉；點碗放入 |
| 灑配料 | 點 **蔥花 / 香菜 / 辣椒**，在碗上方點或拖 | 配料像小花灑進碗裡（花點只在碗上方顯示） |

- **即時換手**：拿著任何東西時，點別的料理站會**立刻換成新的**，不必先放下。
- **火候即技巧**：撈麵的時機決定星級——剛剛好 3 星，太生 / 過頭 2 星。這是整局唯一的手感技巧點。

### 成功 / 失敗

- **可上菜**：湯 + 麵 + 牛肉齊全，且配料與訂單**完全一致** → 浮出「上菜」。
- **無法挽回**：灑了訂單**不要**的配料（配料灑下去拿不掉）→ 浮出「倒掉」重做。
- 基礎料（湯 / 麵 / 牛肉）沒放是可補救的，不算錯。

## 四、場景與介面

### 啟動畫面 / 菜單（櫃台）

- 一整面**收銀台場景**：綠色暖簾招牌「拉麵怪奇物語」、紅燈籠、牆上價目板、收銀機、招財貓、木質櫃台。
- 暖簾 / 牆 / 櫃台三段色帶**程式化全寬繪製**，隨螢幕無限延展。
- **遊戲啟動圖（boot splash）就是這個櫃台畫面**，只是沒有按鈕——載入與菜單觀感一致。
- 操作：**點畫面任意處即可開始**（手機遊戲，不提 Enter）。

### 做麵畫面（直式佈局，由上而下）

1. **HUD**：店名「拉麵屋」、完成單數、「← 選單」返回鍵（貼著螢幕頂，避開瀏海）。
2. **訂單紙條**：今日訂單 / 牛肉麵 + 客人想要的配料色塊。
3. **組裝碗**：放在抬高的小木台上，居中。
4. **小料台**：緊貼碗台下方的長木台，擺著牛肉 / 蔥花 / 香菜 / 辣椒四個**陶瓷碗**。
5. **湯鍋 / 麵鍋**：畫面最下方兩個**寬陶瓷碗**——湯鍋是**黃油湯**（冒泡），麵鍋是**清水**（煮麵時浮出麵團）。鍋身在畫面外、只露碗口，各冒一縷小蒸汽。
6. **動作按鈕**：「倒掉 / 上菜」——預設不顯示，按情境浮出，貼著螢幕底（避開 Home Indicator）。

### 勝利結算

暗色彈窗（畫面正中）：「上菜成功！」+ 成品碗預覽 + 三顆星評分 + 評語 + 「回主選單 / 再來一單」。

## 五、美術風格

- **美式卡通**：粗黑描邊、平塗鮮豔色、簡單高光，像貼紙 / 表情包。
- **像素繁體中文字**：採用 [Zpix（最像素）](https://github.com/SolidZORO/zpix-pixel-font) 點陣字，與卡通插圖形成「平滑圖 + 像素字」的對比質感。
- **線性過濾**：高 DPI 手機上放大不鋸齒（曲線平滑）。
- **App 圖標 / 啟動**：一碗冒著兩縷平行蒸汽的拉麵。

### 美術管線

所有插圖都是**手寫 SVG → PNG**，用 Python `cairosvg` 在建置前產生：

```
assets/cook/gen_svg.py    # 碗、湯、麵、牛肉、湯勺、筷子、小料碗、湯/麵碗…
assets/env/gen_svg.py     # 收銀台場景道具（燈籠、價目板、收銀機、招財貓）
assets/icon/gen_svg.py    # App 圖標 + iOS 各尺寸
```

字體採用**完整 Zpix**，任何繁體字直接可用，不需逐字子集化。

## 六、技術架構

| 項目 | 內容 |
|---|---|
| 引擎 | Godot **4.3** |
| 語言 | GDScript |
| 算繪 | **GL Compatibility**（桌面 + 行動） |
| 基準解析度 | 直式 **270 × 480**，`stretch = canvas_items` / `expand` |
| 響應式 | 內容置中，外層木桌 / 牆面**無限鋪滿**任意比例；HUD 貼頂、動作按鈕貼底；iOS 安全區（瀏海 / Home Indicator）內縮 |
| 繪製方式 | `Node2D._draw()` 程式化繪圖 + SVG 精靈 |
| 存檔 | `user://` JSON（累計完成單數） |

## 七、部署

- **網頁**：推送 `master` 觸發 GitHub Actions，匯出 Web 後部署到 GitHub Pages。
  匯出資源檔名加上 commit SHA 做**快取破壞**，避免 CDN 發舊版；載入頁背景用菜單色帶鋪滿。
- **iOS / TestFlight**：手動觸發 `ios.yml`（macOS runner + Xcode 26 SDK），Godot 匯出已簽章 `.ipa`，`altool` 上傳 TestFlight；版號等元資料用 [`asc`](https://github.com/rork) CLI 管理。
  - App 名稱：**拉麵怪奇物語**（CFBundleDisplayName + App Store Connect）
  - 朝向：強制**直式**（`display/window/handheld/orientation = 1`）
