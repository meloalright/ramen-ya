# 拉麵怪奇物語 — 遊戲設計理念

> 一款純手作視角、一次只做一碗的直式手機拉麵小品。
> Made with Godot 4.3 ・ 線上試玩：<https://meloalright.github.io/ramen-ya/>

---

## 一、一句話概念

**「親手做一碗拉麵」**——不經營、不排隊、不數錢，只專注於把眼前這一碗麵做好。
玩家站在拉麵店的料理台後，從湯鍋舀湯、從麵鍋撈麵、灑上客人要的配料，做對了就完成。

整個體驗追求**輕、爽、手感**：拿起、放下、灑料、完成，每一下點擊都立刻有回饋與音效，沒有多餘的選單與數值。

---

## 二、核心理念

1. **手作感優先**
   每個動作都對應一個真實的料理工具：舀湯是一把**薄壁不鏽鋼大湯勺**（湯只在勺口露一面）、撈麵是一雙夾著**長麵線**的大筷子。游標拿著湯 / 麵時看得到工具；牛肉與小料則「拿著不顯示、放進碗才出現」。

2. **一次一單，做完即結算**
   不做排隊經營。畫面上永遠只有一張訂單，做對就完成、彈出勝利結算，乾淨俐落。

3. **介面只在需要時出現**
   預設畫面上沒有任何動作按鈕。
   - 當碗符合訂單 → 才浮出「完成」。
   - 當灑錯了無法挽回的配料 → 才浮出「倒掉」。
   讓玩家專注在「做」，而不是「找按鈕」。

4. **場景即介面，背景無限延展**
   料理台的木桌、菜單頁的牆面 / 櫃台，都是程式化繪製、隨螢幕無限鋪滿，任何比例的手機 / 桌面都不留黑邊。菜單牆面還打著一道**斜照光**——中間亮、兩邊暗。

5. **可把玩的菜單**
   菜單牆上的**版本貼紙**與一朵**粉色小花**都能**拿起、拖到牆面任意處放下**（有拿起 / 放下音效，位置**永久存檔**；重置資料時回到初始位置）。是一個小小的「整理自己店面」的趣味。

## 三、玩法循環

```
菜單 → 點「開始」 → 做麵 → 製作成功 → 勝利結算 → 再來一單
```

### 做一碗麵的步驟

| 步驟 | 操作 | 視覺回饋 |
|---|---|---|
| 下湯 | 點下方**湯鍋** | 游標提起一把盛著**黃油湯**的薄壁鋼勺；點碗倒入 |
| 煮麵 | 點下方**麵鍋**（第一次） | 麵下鍋開始煮，鍋裡浮出一團**麵線**，旁邊出現**火候條** |
| 撈麵 | 再點一次麵鍋 | 游標提起一雙夾著**長麵線**的大筷子；點碗放入 |
| 加牛肉 | 點**牛肉**小料碗 | 移到碗上方放入（無游標攜帶物） |
| 灑配料 | 點 **蔥花 / 香菜 / 辣椒**，在碗上方點或拖 | 配料灑進碗裡（無游標攜帶物，只在碗裡出現） |

- **即時換手**：拿著任何東西時，點別的料理站會**立刻換成新的**，不必先放下。
- **火候即技巧**：撈麵的時機決定星級——剛剛好 3 星，太生 / 過頭 2 星。這是整局唯一的手感技巧點。

### 成功 / 失敗

- **可完成**：湯 + 麵 + 牛肉齊全，且配料與訂單**完全一致** → 浮出「完成」。
- **無法挽回**：灑了訂單**不要**的配料（配料灑下去拿不掉）→ 浮出「倒掉」重做。
- 基礎料（湯 / 麵 / 牛肉）沒放是可補救的，不算錯。

## 四、場景與介面

### 啟動畫面

- **黑底 + 圓形遊戲 logo**（一碗拉麵切成圓形），網頁載入頁同款，乾淨聚焦。

### 菜單

- **牆面光影**：米色牆面打著一道斜照光，**中間亮、兩邊暗**（暗角 + 由上而下的暖光帶）。
- **店名**：「拉麵怪奇物語」以**湯色（黃油湯色）**顯示，落在**頂部安全區與桌面之間**的**黃金比**位置（靠上 0.618）。
- **牆上道具**：一朵**粉色小花**與一張**白紙版本貼紙**（iOS 圓角 app logo + 版本號，微微歪斜貼著），兩者皆可**拖拽**並存檔。
- **櫃台**：一台黃色**共享充電寶租借機**坐在木質櫃台上；木桌無限延展。
- **層級**：牆面 1 < 貼紙 2 < 花 5 < 桌子 20 < 充電寶機 30，前後關係明確。
- 操作：點「**開始**」按鈕進入（手機遊戲，不提 Enter；點別處不會誤觸開始）。
- **隱藏彩蛋**：連點充電寶機 **7 下**，彈出確認框可**重置本地資料**（完成數、擺放、存檔；彈窗只認自己的按鈕）。

### 做麵畫面（直式佈局，由上而下）

1. **HUD**：左上「← 選單」返回鍵、右上**本輪完成數**（「本輪 N 單」，本輪 0 單時不顯示，與選單的**累計總數**「已完成 N 單」是兩個概念），兩者共用同一條中線（貼著螢幕頂，避開瀏海）。
2. **訂單紙條**：今日訂單 / 牛肉麵 + 客人想要的配料色塊。
3. **組裝碗**：放在抬高的小木台上，居中。
4. **小料台**：碗台下方的木台，擺著牛肉 / 蔥花 / 香菜 / 辣椒四個**陶瓷碗**。
5. **湯鍋 / 麵鍋**：畫面最下方兩個**寬陶瓷碗**——湯鍋是**黃油湯**（冒泡），麵鍋是**清水**（煮麵時浮出麵線）。鍋身在畫面外、只露碗口，各冒一縷小蒸汽。
6. **動作按鈕**：「倒掉 / 完成」——預設不顯示，按情境浮出，貼著螢幕底（避開 Home Indicator）。

### 勝利結算

暗色彈窗（畫面正中）：「**製作成功**」+ **剛做好的真實成品碗**（含玩家實際撒下的配料，依原位置等比呈現）+ 三顆星評分 + 評語 + 「回主選單 / 再來一單」。彈窗是**模態**的——蒙層連左上角的返回鍵都一併蓋住、停用，只認自己的兩個按鈕，點別處不會關閉或重開。

## 五、美術風格

- **美式卡通**：粗黑描邊、平塗鮮豔色、簡單高光，像貼紙 / 表情包。
- **像素繁體中文字**：採用 [Zpix（最像素）](https://github.com/SolidZORO/zpix-pixel-font) 點陣字，與卡通插圖形成「平滑圖 + 像素字」的對比質感。完整字庫，任何繁體字直接可用。
- **超採樣 + 線性過濾**：所有 SVG 精靈以 **4× 設計尺寸**柵格化再縮小顯示，高 DPI 手機上曲線依然銳利不糊。
- **App 圖標 / 啟動**：一碗冒著兩縷平行蒸汽的拉麵。

### 美術管線

所有插圖都是**手寫 SVG → PNG**，用 Python `cairosvg` 在建置前產生：

```
assets/cook/gen_svg.py    # 碗、湯、麵線、牛肉、薄壁鋼湯勺、筷子、小料碗、湯/麵碗…
assets/env/gen_svg.py     # 菜單道具：共享充電寶租借機（cashier.png）、價目板（board.png）
assets/icon/gen_svg.py    # App 圖標 + iOS 各尺寸；圓形 / iOS 圓角 logo 由圖標派生
```

## 六、技術架構

| 項目 | 內容 |
|---|---|
| 引擎 | Godot **4.3** |
| 語言 | GDScript |
| 算繪 | **GL Compatibility**（桌面 + 行動） |
| 基準解析度 | 直式 **270 × 480**，`stretch = canvas_items` / `expand` |
| 響應式 | 內容置中，外層木桌 / 牆面 / 花簇**無限鋪滿**任意比例；HUD 貼頂、動作按鈕貼底；iOS 安全區（瀏海 / Home Indicator）內縮 |
| 繪製方式 | `Node2D._draw()` 程式化繪圖 + 4× 超採樣 SVG 精靈 |
| 文字排版 | 自製置中：依末字是否為全形字補正「字尾留白」，混排（中文 + 數字 + 標點）也精準居中 |
| 音效 | 持久的 `Music` autoload：BGM + 按鈕點擊音 + 菜單拖拽的拿起 / 放下音 |
| 存檔 | `user://` JSON：累計完成單數 **＋ 牆面道具擺放位置** |

## 七、部署

- **網頁**：推送 `master` 觸發 GitHub Actions，匯出 Web 後部署到 GitHub Pages。
  匯出資源檔名加上 commit SHA 做**快取破壞**，避免 CDN 發舊版；載入頁為黑底圓形 logo。
- **iOS / TestFlight**：手動觸發 `ios.yml`（macOS runner + Xcode 26 SDK），Godot 匯出已簽章 `.ipa`，`altool` 上傳 TestFlight；版號 / 審核等元資料用 [`asc`](https://github.com/rork) CLI 管理。
  - App 名稱：**拉麵怪奇物語**（CFBundleDisplayName + App Store Connect）
  - 朝向：強制**直式**（`display/window/handheld/orientation = 1`）
  - 顯示：隱藏系統狀態欄、全螢幕（`UIStatusBarHidden = true`）
